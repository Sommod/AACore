<h1>What is this?</h1>
This plugin is aimed to be a massive plugin that implements many key features that plugins use and
both simplify and broaden the options that Bukkit/Spigot has for their API system. Many keys features
that are represented within this plugin are similar to that of the commonly know <b><i>Vault</i></b> and
<b><i>ProtocoLib</i></b> within one plugin as well as other aspects like command handling, Better YML
configuration and other features.
<h1>What this is not</h1>
This is not a replacement for any plugin(s) that already exist, this is to only help and supply more
features and control over the functions that may not be covered in other plugins. This does have similar
items within and directly constitutes to ProtocoLib, but tries to simplify such actions for newer and
unfamiliar Developers.
<h1>How to implement</h1>
This plugin only contains a few files that are not abstract or interface-type classes. Most of the class
within this plugin are used as parent classes to help organize and to allow access to various other actions.
To add, this can simply be referenced by the non-maven way of implementing this into the Build Path of your
project. After that, you can extend the classes found within the <b>API</b> portion of the plugin.
<h1>Version Requirements</h1>
This plugin is set in Spigot-1.16.1.RO-1. All classes that deal with version-specific methods and functions
are accounted within their file(s) or are converted via the <b>Converter</b> class within the API. Any version
that is of 1.16.1 or higher will work without issue, anything prior to the 1.16.1 update may not work or cause
errors. If you use a prior version, then ensure that the class(es) are compatible with the server version type.
